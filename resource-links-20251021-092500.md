## 2. Python Package Manager (uv)

### uv Documentation
- **Official Site**: https://docs.astral.sh/uv/
- **Tutorial**: https://www.datacamp.com/tutorial/python-uv
- **Installation**: https://github.com/astral-sh/uv
- **Speed**: 10-100x faster than pip, 80x faster than venv

### Key Commands
```bash
# Install uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# Create virtual environment
uv venv

# Activate environment
source .venv/bin/activate  # Unix
.venv\\Scripts\\activate    # Windows

# Install dependencies
uv pip install -r requirements.txt
uv pip install -e ".[dev]"

# Sync dependencies
uv sync
```

**Resource**: https://realpython.com/python-uv/

## 8. Forum Gamification Best Practices

### Research Articles
- **Points Systems**: https://gamificationnation.com/blog/gamification-mechanic-monday-point-systems/
- **Behavioral Economics**: https://www.smartico.ai/blog-post/gamification-behavioral-economics
- **Anti-Gaming**: https://www.frontiersin.org/journals/education/articles/10.3389/feduc.2023.1212994/full

### Existing Platform Examples

#### Stack Overflow Reputation System
- **How It Works**: https://stackoverflow.com/help/whats-reputation
- **Privileges**: https://stackoverflow.com/help/privileges
- **Key Insights**:
  - +10 points per upvote on answer
  - +10 points per upvote on question (changed from 5 in 2019)
  - Daily cap: +200 reputation max
  - Reputation unlocks privileges at thresholds

#### Reddit Karma System
- **How It Works**: Upvotes/downvotes on posts and comments
- **No Monetary Value**: Purely reputation-based
- **User Levels**: Based on account age + karma for moderation

### Design Principles
1. **Multiple Point Types**: Experience points vs redeemable points
2. **Anti-Inflation**: Burn mechanisms, spending > earning
3. **Cultural Fit**: European scale (10-based) vs US (100-based)
4. **Fairness**: Prevent free-rider problems in team rewards
5. **Engagement**: Balance competition with collaboration

## 11. Deployment Platforms

### Vercel (Recommended for Serverless)
- **Official Site**: https://vercel.com/
- **Python Support**: https://vercel.com/docs/functions/runtimes/python
- **Free Tier**: Generous for hobby projects
- **Limitations**: No long-running processes, limited build minutes

### Railway (Recommended for Full Control)
- **Official Site**: https://railway.app/
- **Free Tier**: $5 credit signup, free under $5/month
- **Features**: PostgreSQL, Redis, persistent storage
- **Python Deployment**: https://docs.railway.app/guides/python

### Render
- **Official Site**: https://render.com/
- **Free Tier**: 750 hours/month for one service
- **Features**: Auto-scaling, TLS certificates, Docker support

### PythonAnywhere
- **Official Site**: https://www.pythonanywhere.com/
- **Free Tier**: 1 web app, MySQL database
- **Limitations**: No custom domains, limited external API access

### Comparison Table
| Platform | Free Tier | PostgreSQL | Redis | WebSockets | Best For |
|----------|-----------|------------|-------|------------|----------|
| Vercel | Yes | Via Supabase | No | Limited | Static sites, APIs |
| Railway | $5 credit | Yes | Yes | Yes | Full-stack apps |
| Render | 750hrs/mo | Yes | Yes | Yes | Production apps |
| PythonAnywhere | 1 app | MySQL only | No | No | Simple apps |

**Decision**: Railway for MVP, migrate to dedicated hosting later
